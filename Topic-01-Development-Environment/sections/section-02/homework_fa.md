**زمان تخمینی**: ۱۲۰-۱۵۰ دقیقه

## 📋 نمای کلی تمرین

در این تمرین، عملیات مخزن و همکاری از راه دور را به تسلط خواهید رساند. مخزن‌ها را به صورت محلی و روی گیت‌هاب ایجاد خواهید کرد، یاد خواهید گرفت که پروژه‌های موجود را کلون کنید، اتصالات از راه دور را مدیریت کنید و گردش کار کامل کنترل نسخه توزیع‌شده را درک کنید. قدرت توزیع‌شدگی گیت را از طریق عملیات عملی از راه دور تجربه خواهید کرد.

**اهداف یادگیری:**
- ایجاد و کلون کردن مخزن‌های گیت (محلی و از راه دور)
- مدیریت اتصالات مخزن از راه دور
- درک عملیات push/pull/fetch
- کار با گیت‌هاب به عنوان میزبان از راه دور
- حفظ سلامت و پیکربندی مخزن
- تجربه گردش کار کنترل نسخه توزیع‌شده

## 🎯 الزامات

### قسمت ۱: ایجاد مخزن و راه‌اندازی از راه دور (۲۵ امتیاز)

#### وظیفه ۱.۱: ایجاد مخزن گیت‌هاب
**هدف**: راه‌اندازی مخزن حرفه‌ای روی گیت‌هاب برای همکاری

**الزامات:**
- ایجاد مخزن جدید روی گیت‌هاب
- انتخاب نام توصیفی با پیروی از قراردادها: `repository-operations-practice`
- اضافه کردن توضیح واضح: "مخزن تمرین برای عملیات از راه دور گیت"
- عمومی کردن مخزن
- راه‌اندازی با فایل README.md
- اضافه کردن موضوعات مرتبط: `git`، `version-control`، `tutorial`

**راه‌اندازی مخزن:**
```bash
# مخزن ابتدا باید از طریق رابط وب گیت‌هاب ایجاد شود
# سپس آن را به صورت محلی کلون کنید
git clone https://github.com/yourusername/repository-operations-practice.git
cd repository-operations-practice
```

**تحویل‌دادنی‌ها:**
- اسکرین‌شات ایجاد مخزن گیت‌هاب
- URL مخزن و دستور کلون استفاده شده
- ساختار اولیه مخزن

#### وظیفه ۱.۲: پیکربندی مخزن محلی
**هدف**: پیکربندی مخزن محلی خود برای همکاری از راه دور

**الزامات:**
- تأیید تنظیم صحیح اتصالات از راه دور
- پیکربندی تنظیمات مخصوص مخزن در صورت نیاز
- درک شاخه‌های پیگیری از راه دور
- تنظیم ترجیحات شاخه پیش‌فرض

**پیکربندی از راه دور:**
```bash
# بررسی پیکربندی از راه دور
git remote -v

# تأیید URLها و نام‌های از راه دور
git remote show origin

# بررسی پیگیری شاخه
git branch -a
git status
```

**تحویل‌دادنی‌ها:**
- خروجی `git remote -v` و `git remote show origin`
- اسکرین‌شات اطلاعات پیگیری شاخه
- تأیید تنظیم صحیح از راه دور

### قسمت ۲: عملیات مخزن و مدیریت فایل‌ها (۳۰ امتیاز)

#### وظیفه ۲.۱: پر کردن مخزن با محتوا
**هدف**: اضافه کردن فایل‌های پروژه و ایجاد تاریخچه مخزن

**الزامات:**
- ایجاد ساختار پروژه با چندین فایل و دایرکتوری
- اضافه کردن فایل‌ها در کامیت‌های منطقی
- ساخت تاریخچه پیشرفت به صورت تدریجی
- نشان دادن درک زمان کامیت کردن

**ایجاد ساختار پروژه:**
```bash
# ایجاد ساختار پروژه
mkdir -p src utils docs examples

# ایجاد فایل‌های اولیه
echo "# تمرین عملیات مخزن" > README.md
echo "print('سلام از عملیات مخزن!')" > src/main.py
echo "# توابع کاربردی" > utils/helpers.py
echo "# مستندات پروژه" > docs/overview.md

# بررسی وضعیت و اضافه کردن فایل‌ها
git status
git add README.md
git commit -m "اضافه کردن README پروژه و ساختار اولیه"

# اضافه کردن کد منبع
git add src/
git commit -m "اضافه کردن دایرکتوری کد منبع و اسکریپت اصلی"

# اضافه کردن ابزارها
git add utils/
git commit -m "اضافه کردن توابع کاربردی و ابزارهای کمکی"
```

**تحویل‌دادنی‌ها:**
- مخزن با سازماندهی منطقی فایل‌ها
- حداقل ۴ کامیت نشان‌دهنده پیشرفت توسعه
- `git log --oneline` نشان‌دهنده پیشرفت کامیت‌ها

#### وظیفه ۲.۲: عملیات push از راه دور
**هدف**: یادگیری push کردن تغییرات محلی به مخزن‌های از راه دور

**الزامات:**
- push کردن کامیت‌ها به گیت‌هاب
- رسیدگی به سناریوهای مختلف push (push اولیه، pushهای بعدی)
- درک شاخه‌های پیگیری
- تأیید ظاهر شدن تغییرات روی گیت‌هاب

**عملیات push:**
```bash
# push به از راه دور (اولین بار)
git push -u origin main

# بررسی وضعیت از راه دور
git status

# ایجاد تغییرات بیشتر و push دوباره
echo "مستندات اضافی" >> docs/overview.md
git add docs/
git commit -m "اضافه کردن مستندات پروژه"
git push

# تأیید روی گیت‌هاب
# بررسی اینکه کامیت‌ها و فایل‌ها روی رابط وب گیت‌هاب ظاهر شده‌اند
```

**تحویل‌دادنی‌ها:**
- عملیات push موفق با اسکرین‌شات‌ها
- مخزن گیت‌هاب نشان‌دهنده فایل‌های کامیت شده
- تأیید همگام‌سازی از راه دور

### قسمت ۳: کلون کردن و گردش کار همکاری (۲۵ امتیاز)

#### وظیفه ۳.۱: کلون کردن مخزن موجود
**هدف**: تجربه کلون کردن مخزن‌ها و کار با کدبیس‌های موجود

**الزامات:**
- کلون کردن مخزن موجود (مخزن آموزشی عمومی یا ایجاد مخزن دوم استفاده کنید)
- درک تفاوت بین کلون کردن و فورک کردن
- کار با مخزن کلون شده به صورت مستقل
- ایجاد و کامیت کردن تغییرات در مخزن کلون شده

**عملیات کلون:**
```bash
# کلون کردن مخزن خود برای شبیه‌سازی گردش کار همکاری
cd ..
git clone https://github.com/yourusername/repository-operations-practice.git cloned-repo
cd cloned-repo

# بررسی داشتن همه فایل‌ها و تاریخچه
ls -la
git log --oneline

# ایجاد تغییرات در مخزن کلون شده
echo "تغییرات از دیدگاه مخزن کلون شده" > cloned-changes.txt
git add cloned-changes.txt
git commit -m "اضافه کردن تغییرات از دیدگاه مخزن کلون شده"

# push کردن تغییرات
git push origin main
```

**تحویل‌دادنی‌ها:**
- کلون موفق مخزن با تاریخچه کامل
- تغییرات ایجاد و کامیت شده در مخزن کلون شده
- تأیید push از مخزن کلون شده

#### وظیفه ۳.۲: درک عملیات از راه دور
**هدف**: یادگیری تفاوت fetch در برابر pull و مدیریت شاخه از راه دور

**الزامات:**
- درک تفاوت بین fetch و pull
- تمرین fetch کردن تغییرات بدون merge
- یادگیری عملیات شاخه از راه دور
- درک شاخه‌های پیگیری از راه دور

**تمرین عملیات از راه دور:**
```bash
# بازگشت به مخزن اصلی
cd ../repository-operations-practice

# fetch کردن تغییرات از از راه دور (دانلود بدون merge)
git fetch origin

# دیدن آنچه تغییر کرده
git log --oneline main..origin/main

# pull کردن تغییرات (fetch + merge)
git pull origin main

# بررسی وضعیت و تأیید همگام‌سازی
git status
git log --oneline --graph --all
```

**تحویل‌دادنی‌ها:**
- نمایش عملیات fetch در برابر pull
- درک همگام‌سازی شاخه از راه دور
- تأیید وضعیت مخزن پس از عملیات

### قسمت ۴: نگهداری مخزن و تحلیل (۲۰ امتیاز)

#### وظیفه ۴.۱: سلامت و نگهداری مخزن
**هدف**: یادگیری نگهداری و بهینه‌سازی مخزن‌های گیت

**الزامات:**
- بررسی سلامت و یکپارچگی مخزن
- انجام عملیات نگهداری مخزن
- درک اندازه مخزن و بهینه‌سازی
- پاکسازی فایل‌ها و دایرکتوری‌های غیرضروری

**نگهداری مخزن:**
```bash
# بررسی سلامت مخزن
git fsck

# بررسی اندازه مخزن و آمار
git count-objects -v

# بهینه‌سازی مخزن (فشرده‌سازی آبجکت‌ها)
git gc --aggressive

# بررسی فایل‌های بزرگ یا مسائل
git verify-pack -v .git/objects/pack/*.idx | sort -k 3 -n | tail -5

# پاکسازی فایل‌های untracked (مراقب باشید!)
git clean -n  # پیش‌نمایش آنچه حذف خواهد شد
```

**تحویل‌دادنی‌ها:**
- نتایج بررسی سلامت مخزن
- آمار قبل/بعد بهینه‌سازی
- درک عملیات نگهداری

#### وظیفه ۴.۲: تحلیل عملیات مخزن و خلاصه
**هدف**: تحلیل عملیات مخزن و مستندسازی یادگیری

**الزامات:**
- مستندسازی تمام عملیات مخزن انجام شده
- تحلیل تفاوت بین عملیات محلی و از راه دور
- درک مزایای کنترل نسخه توزیع‌شده
- تأمل در مورد چالش‌های گردش کار همکاری

**تحلیل مخزن:**
- لاگ کامل عملیات (کلون، push، pull، fetch)
- درک مخزن محلی در برابر از راه دور
- مزایای کنترل نسخه توزیع‌شده تجربه شده
- چالش‌ها و راه‌حل‌ها در گردش کار همکاری

**تحویل‌دادنی‌ها:**
- لاگ جامع عملیات مخزن
- تحلیل گردش کار همکاری از راه دور
- بینش‌های شخصی در مورد کنترل نسخه توزیع‌شده
